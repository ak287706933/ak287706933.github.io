语义化标签

  • 在页面少加一些文本，避免影响文本的权重
  • footer用Js加“|”
  • 作用：
    • 能够便于开发者阅读和写出更优雅的代码，代码如诗
    • 让浏览器或是网络爬虫可以更好的解析，从而更更好的分析其中的内容，更好的搜索引擎优化
  • HTML的职责是描述一块内容是什么（或者意义）

新标签的兼容问题

应用程序标签——具有交互作用的标签

  • DataList（数据列表）

    • 文件不建议以文件路径打开，sublime server的插件，让文件以HTTP服务的形式打开，默认使用800端口》》以localhost开头打开（不要点stop，直接退出）

    • 数据列表想要呈现出来需要一个载体（input），可以有自动匹配，不能修改样式，不可以自定义的不建议使用，jQuery的体验更好

  • Progress（进度条）

    • setting-element-show user agent shadow DOM 勾选

    • 控件有自己的默认样式，input/下拉框 等内置控件内部由一个虚拟的DOM铺垫

    • -webkit-appearance：none；就可以设置宽高等

    • 伪对象 例如a:：after

    • .progress::-webkit-progress-bar

  • Meter（数据显示器）

    • <meta min="0" max="100" low="40" high="90" optimum="100" value="91"></meter>兼容性不好

  • Menu、Details（不兼容，不做深入了解）

  • zh-CN 简体中文 zh-TW 繁体中文

新属性

  • 不兼容的浏览器对于不认识的标签当成是div解析，极个别会报错

  • 解析标题之前，在header写入或者script写在body最下面，不要影响页面渲染

  • 结构数据标记

    • 很高级，只有google支持

  • ARIA属性

    • Accessible Rich Internet Application（无障碍富互联网应用程序）

    • 主要针对于屏幕阅读设备（eg.NVDA)，更好更快地理解网页

    • label给input做标注

    • WAI-ARIA无障碍网页应用属性完全展示

    • 通过设置role属性设置 role=”navigation” role=”option”

    • 通过设置aria属性设置关系

  • 自定义属性 data-

    • 对页面没有侵入，不影响功能
    • 通过DOM存储与DOM对象强相关的数据

  • innerText方法不兼容，不建议使用

  • 建议不给没有交互功能的标签添加事件

box.webkitRequestFullScreen();

document.webkitCancelFullScreen();

链接关系

  • 通过链接关系属性声明连入的链接和当前文档之间的关系，便于机器理解文档

  • <a href="01-application-tags.html" rel="prev">上一页</a>

  • <a href="02-XXX.html">下一页</a>

  • 链接关系表

    • alternate 文档的可选版本（例如打印页、翻译页或镜像）
    • friend 友情链接
    • nofollow 用于指定google搜索引擎不要跟踪链接

智能表单

  • 新的表单类型，不建议在网页中使用
  • 可以针对需求使用特定的只能表单类型
    • <form action="#" method=""></form>
    • 之间学过的表单标签：file text password radio checkbox image button submit button reset select
    • button元素和type=“button”
    • <input type="datetime">兼容性不佳，可以用<input type="date">不能在PC上用，样式无法自定义
    • <input type="color">在移动端不支持
    • <input type="range" min="1" max="10" step="5" value="5"> 定义带有 slider 控件的数字字段。value：默认 step：单步选择
    • <input type="email" value="some@qq.com" pattern="正则表达式">本身具备校验
    • <input type="text" required>必填选项，常用于移动端，无法设置样式
    • <input type="tel" placeholder="(555)555-5555" pattern="">本身不具备校验
    • input：range + tab 快捷键

虚拟键盘适配

  • IOS模拟器 simulator 默认不弹出键盘 在IOS上 Ctrl+K

网络多媒体

  • <audio src="路径" controls="controls"></audio> 默认不显示播放控件

  • cd+d ctrl+K

  • jquery注册时间都用onclick

  • zepto用于移动端，要求轻，压缩后文件很小

  • 陌生的API在console.log 输入audio

  • dash 可以下载文档，检索速度非常快

  • 视频

    • <video src="" controls="controls"></video>
    • 每一种视频格式是有版权的，不同浏览器支持的文件的格式不一样
    • 通过source的标签定义视频源
    • <video controls="controls" poster="top.png" autoplay="autoplay"><source src="" type="video.ogg"><source src="" type="video.mp4">支持video元素文字可以显示，如果不支持会显示文字</video>
    • 如果不支持用flash
    • poster: 视频的海报图，常用视频的第一帧
    • autoplay：autoplay 自动播放
    • muted: muted 是否静音
    • loop: loop 是否循环
    • preload 是否预加载

  • 兼容模式：https://html5media.info/ 组件

  • 字幕兼容性不好

  • <track lable="" kind="" src="">

SVG

  • Scalable Vector Graphics：可缩放矢量图形

  • 位图就是由一堆像素点组成，由有颜色的像素

  • 矢量图记录的是线，路径，填充色，任意缩放，图片不会失真，基于可扩展标记语言

  • 文档标题<?xml version="1.0" encoding="utf-8"……> 为xml文档

  • 优点：体积小（数据量少），质量高（放大无锯齿），效果好，可控程度高

  • 定义 线性渐变 stop circle 定义了一个圆 定义了url(#id名)

  • 可以用CSS填充颜色

  • 图标字体

  • 以文件的方式使用 <iframe src="demo.svg" frameholder="0"></iframe> 推荐使用，不能通过CSS控制样式

  • <object data="demo.svg" type=""></object> 在标准中有定义

  • <embed src="demo.svg" type=""> 在标准中未定义，但浏览器都支持

  • <svg data-src="demo.src"></svg> 浏览器不支持，学了异步之后

  • 引入SVG文件后用CSS控制颜色

  • 用于高清的logo图，图标

  • 可以在网页中使用别人做好的SVG；可以对SVG文件做出基本调整；会用样式修饰SVG

  • iframe .svg本身是个文档，可以通过iframe载入进来，无法改别人的样式

svg文件的基本使用

  • https的默认端口是443
  • http的默认端口是800
  • ajax直接可以修改样式
