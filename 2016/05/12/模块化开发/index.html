模块化开发

有哪些模块化方法

1、全局函数

以计算器为例子

缺点:严格来说，这并不是真正的模块化，我们只能人为的通过注释的方式来标明这四个方法是属于一个计算相关的模块。

2、封装成对象

缺点:暴露了全局对象，还是可能冲突，命名空间越来越长，越写越复杂。

3、匿名函数自执行形式（沙箱形式）

缺点:相对于封装成对象，沙箱模式已经很优秀了，但还是暴露了全局对象比如add,substract···这四个方法。

模块化的规范

浏览器端规范: sea.js require.js
服务器端规范: node.js
跨浏览器、服务器的模块规范: CommonJS ECMAScript2015(ES6)

sea.js的使用方法

a) html文件中引入sea.js库
b) 使用seajs.use加载主模块（也叫入口模块），起名一般叫main.js
c) 模块的定义方法
在我们的sea.js当中一个模块就是一个文件
调用define函数定义主模块
用require函数加载其他第三方模块
用module.exports或exports来暴露当前模块的东西
需要注意的点:
1.define(function(require,exports,module){});这里面的require,exports,module三个参数的名字是固定写法，不能简写，顺序不能错了 记忆方法：rem –> 依赖注入
2.如果define里的参数想省略，只能从后面开始省略，只要写了后面的，前面的必须要写上 –> 推荐写全，不要省
3.如果一个模块既不需要公开成员变量，也不需要依赖第三方模块,那么可以全部省略
4.seajs.use(“入口模块”)可以加回调函数，形式如：seajs.use(“入口模块”,callback);

seajs的路径问题

1、在sea.js当中，js/main和./js/main的含义不是一样的
a) js/main代表的是在sea.js所在的文件为基准去找，所以找到的是 –> seajs库所在的父级文件夹/js/main.js 这样的话会找不到的
b) ./js/main代表的是以当前html文件去找
c) seajs.use(),require(“xxx”) –> 其实这里的xxx叫模块标识，不是路径
2、config的作用就是为了简化调用模块
a) base 设置路径 默认值是seajs库文件所在的父文件夹
b) alias 设置模块的别名，简化调用

sea.js之module.exports与exports辨析

  • 本质上用的还是module.exports这个对象，exports的值是module.exports 的引用地址，通过这种方式，我们可以少写一个”module.”
  • module.exports 指向新的对象时，exports 将会断开了与 module.exports 的引用

sea.js加载流程理解

seajs是预加载懒执行。
预加载就是预先把所有的模块全部加载完，懒执行就是需要执行哪块才执行哪块

  • html文件加载主模块
  • 主模块加载依赖的第三方模块
  • 第三方模块如果还有其他的第三方模块依赖，则进一步加载其他第三方模块
